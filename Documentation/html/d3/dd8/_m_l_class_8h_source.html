4 /**
5  *@class KKMLL::MLClass
6  *@brief Represents a "Class" in the Machine Learning Sense.
7  *@author Kurt Kramer
8  *@details
9  *@code
10  **********************************************************************
11  ** MLClass *
12  ** *
13  ** Each instance of this class represents a single Class as used in *
14  ** Machine Learning sense. Each instance of 'FeatureVector' class *
15  ** will point to an instance of this class. There can only be one *
16  ** instance of each Class in memory. Specifically the 'name' field *
17  ** will be unique. This is enforced by making the constructor and *
18  ** destructor private. The only way to create a new instance of a *
19  ** 'MLClass' object is to call one of the static methods of *
20  ** 'CreateNewMLClass'. These methods will look for a instance of *
21  ** 'MLClass' that already exists. If one does not they will then *
22  ** create a new one. *
23  ** *
24  ** Please refer to MLClassList at bottom of file. That class is *
25  ** the object you will most be using when dealing with Images. *
26  ** classes. *
27  ** *
28  ** UmiClass and PicesClass *
29  ** There is a special relationship between this class and a class *
30  ** called 'UmiClass' and 'PicesClass in the 'UnManagedInterface' *
31  ** and PicesInterface libraries, ".net' Managed libraries'. These *
32  ** managed c++ versions of this class. There is a one-to-one *
33  ** correspondence between the two classes. When ever a instance *
34  ** of 'UmiClass' or 'PicesClass' get created they will automatic- *
35  ** ally call the static method 'MLClass::CreateNewMLClass' to get *
36  ** the unmanaged version of the class. *
37  **********************************************************************
38  *@endcode
39  *@see KKMLL::MLClassList, KKMLL::FeatureVector, UnManagedInterface::UmiClass, MLL::PicesClass
40  */
