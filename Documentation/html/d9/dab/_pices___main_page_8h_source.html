5  *@section Introduction
6  *
7  * The Pices system is a set of applications that were developed as part of my (Kurt Kramer) Graduate work. I
8  * worked closely with my major professors Dr. Dmitry Goldgof and Dr. Lawrence Hall and Marine
9  * Scientist Dr. Andrew Remsen. This work was originally done to process the imagery and
10  * instrumentation data collected by the SIPPER imaging platform developed by University of
11  * South Florida Marine Science Center of Oceans Technology. It manages data from cruises dating
12  * back to 2002 through 2014 and is being updated to work with the latest implementation of SIPPER
13  * for future deployments.<br>
14  *
15  * The system consists of several applications and libraries. Most of the code is written in c++
16  * and is meant to be O/S neutral. The GUI functionality is written in .net c#. There is extensive
17  * use of Image Processing (IP) and Machine Learning (ML) techniques used in the processing of data.
18  *
19  *
20  *@section Applications
21  * There are more than 20 applications in the system but only a few comprise the bulk of the
22  * functionality
23  *
24  * List of some but not all applications:
25  * - PicesCommander The main GUI application that provides a thumbnail view of collected plankton images;
26  * Provides charting functionality, classification processes, reporting facilities allowing
27  * user to analyze data by deployments, stations, instrument parameters(O^2, Depth, Salinity,
28  * etc...). Other functionality consists of validation of data, updating of training
29  * libraries(Active Learning), and reclassification of imagery based off user improved
30  * training data. Randomly extract data from deployments for ground-truthing to
31  * help gauge classification performance.
32  *
33  * - WindowsImageExtraction Where raw Sipper data is fed into the system; this application extracts imagery,
34  * Instrumentation data, computes feature vectors m and performs initial classification that
35  * is then fed into the Pices database where the user can then browse the results.
36  *
37  * - SipperFleViewer A simple viewer application that allows the user to view the raw sipper data, both
38  * imagery and instrumentation data.
39  *
40  * - GradeTrainingModel GUI Tool to grade the different training libraries against user ground-truthed data;
41  * generates reports where user can select the classification granularity with respect to
42  * class hierarchy.
43  *
44  * - ImportGPSData Imports GPS data collected by ship into Pices database.
45  *
46  * - SpatialAnalysis
47  *
48  * - CrossValidationApp Application that will perform various flavors of the tradition n-fold cross validation.
49  *
50  * - @link FeatureSelectionApp FeatureSelection @endlink Application that will tune SVM parameters and image extracted features to
51  * maximize classification accuracy; it was decide to run on a cluster; the more
52  * CPU's the faster it processes. It can work in a mixed Windows/Linux environment
53  * only requiring a common file share device.
54  *
55  * - SipperInterface GUI Application that is used to operate the SIPPER III imaging platform.
56  *
57  * - SipperDiskManager - GUI Application that manages the downloading of data from the SIPPER III
58  * Proprietary disk format. The forthcoming SIPPER IV will be using standard
59  * Linux volumes eliminating the need for this application.
60  *
61  *@section Libraries
62  * There are several libraries that are utilized by the various applications.
63  *
64  *List of Libraries:
65  * - @link KKB KKBase(c++) @endlink Basic functionality that is common to other libraries and applications. Some examples
66  * include string management(@link KKB::KKStr KKStr @endlink), Matrix operations(@link KKB::Matrix Matrix @endlink),
67  * Image Processing(@link KKB::Raster Raster @endlink), Token Parsing(@link KKB::Tokenizer Tokenizer @endlink),
68  * Statistics, Histogramming, and common operating system specific routines. Most O/S specific code is implemented in the
69  * module "osServices.cpp" module. Thread management is implemented in the two classes @link KKB::KKThread KKThread @endlink and
70  * @link KKB::KKThreadManager KKThreadManager @endlink. This is an earlier version of the of the KKBase library found
71  * in "KSquareLibraries".
72  *
73  * - @link KKMLL KKMachineLearning(c++) @endlink Classes required to support Machine Learning functionality.
74  * -# @link KKMLL::FeatureFileIO FeatureFileIO @endlink Several common feature data file formats are supported. (sparse, arff,
75  * c45, etc...)
76  * -# @link KKMLL::TrainingConfiguration2 Training-Model-Configuration @endlink class and routines that maintain classifier parameters;
77  * such as classifier type.
78  * -# @link KKMLL::MLClass Machine Learning Classes @endlink and containers for tracking lists of classes.
79  * -# Hierarchical Class naming is supported.
80  * -# @link KKMLL::FeatureVector "FeatureVector" @endlink General purpose container for @link KKMLL::FeatureVector FeatureVector @endlink derived instances.
81  * -# @link KKMLL::FeatureVectorList "FeatureVectorList" @endlink General purpose container for @link KKMLL::FeatureVector FeatureVector @endlink derived instances.
82  * -# @link KKMLL::NormalizationParms Normalization @endlink for FeatureData.
83  * -# @link KKMLL::ConfusionMatrix2 Confusion-Matrix @endlink Object used to capture prediction information and build a confusion matrix.
84  * -# @link KKMLL::CrossValidation CrossValidation @endlink Example 10 fold CV; also (N x X) cross Validation; typically used by grading a classifier.
85  * -# @link KKMLL::Model Training-Model-Implementations @endlink Pair-Wise Feature selected SVM, Common Features SVM, UsfCasCor, and Dual.
86  * -# @link KKMLL::FeatureVectorProducer FeatureVectorProducer @endlink A abstract class used to define the interfaces required for Creating
87  * @link KKMLL::FeatureVector FeatureVector @endlink derived instances from a source object. Examples are
88  * @link KKMLL::GrayScaleImagesFVProducer GrayScaleImagesFVProducer @endlink computes instances of @link KKMLL::GrayScaleImagesFV GrayScaleImagesFV @endlink
89  * feature-vectors; or @link MLL::PicesFVProducer PicesFVProducer @endlink produces a Pices specific feature vector.
90  *
91  * - @link MLL PicesLibary(c++) @endlink Classes and Code that is specific to the Sipper and Pices World Examples are a group of classes meant
92  * to process the different SIPPER file formats going back to the original SIPPER-I platform. A class for MySQL Database interface,
93  * a specialized Pices version of @link KKMLL::TrainingConfiguration2 TrainingConfiguration2 @endlink called PicesTrainigConfiguration.
94  * -# @link MLL::SipperBuff SipperBuff @endlink Base class to the different SIPPER FIle Formats; provides generalized support for processing SIPPER Files.
95  * -# @link MLL::InstrumentData InstrumentData @endlink Maintains the various instrument data fields that are embedded or associated with a SIPPER file such
96  * as Depth, Pith, Roll, GPS Coordinates, Temperature, etc ...
97  * -# @link MLL::InstrumentDataReport InstrumentDataReport @endlink Base class to the different devices that will embed data into
98  * the SIPPER File,such as the Pitch and Roll sensor, CTD, Battery. Each specialization of this class will have the knowledge
99  * on how to interpret/parse the related data stream.
100  * -# @link MLL::InstrumentDataManager InstrumentDataManager @endlink Central reporting facility that all
101  * 'InstrumentDataReport' derived classes send there data thru to be fused together resulting in instances
102  * of 'InstrumentData'.
103  * -# @link MLL::FeatureFileIOPices FeatureFileIOPices @endlink A Pices specific derived class of @link KKMLL::FeatureFileIO FeatureFileIO @endlink
104  * that supports the reading and writing of a Pices specific feature data.
105  * -# @link MLL::PicesTrainingConfiguration PicesTrainingConfiguration @endlink Specialized configuration file for a Pices Training Library.
106  * -# @link MLL::ImageFeatures ImageFeatures @endlink A SIPPER specific implementation of @link KKMLL::FeatureVector FeatureVector @endlink class
107  * that is aware of SIPPER elements such as CTD Instrumentation data, GPS coordinates, Flow-rates, etc...
108  *
109  * - PicesInterface (cli/c++) A mixed Managed/Unmanaged memory model library; its purpose is to provide a interface
110  * from the ".net" world to the unmanaged world of the O/S neutral c++ code.
111  *
112  * - ImageExtractionManager (c++) A frame work for extracting and classifying SIPPER imagery data; it will process
113  * the raw SIPPER data using multiple threads to maximize throughput.
114  *
115  * - @link JobManagment JobManager @endlink Two classes that simplify the implementation of a parallelizable applications. It manages
116  * discrete jobs with their interdependencies. So far only RandomSplits makes use of this library; but
117  * FeatureSelection would be an excellent candidate for utilizing this framework.
118  *
119  * - SipperFile (c#) .net classes used by the different GUI applications.
120  *
121  * - SipperDiskUtilities (c#) Code that interfaces with the Sipper III hard disk. Provides routines that make the
122  * SipperFles contained on the disk to appear as normal binary data files that can be
123  * opened by any .net application.
124  *
125  *
126  *@section OutsideLibraries
127  * I make use of several publicly available libraries:
128  * - <a href="http://www.fftw.org/">FFTW</a> The Fastest Fourier Transform in the West.
129  * - <a href="http://www.zlib.net/">zLib</a> Used to compress images before storing in database.
130  * - <a href="http://www.csie.ntu.edu.tw/~cjlin/libsvm/">LibSVM</a> Support Vector Machine.
131  */
