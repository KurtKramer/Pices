8  ***************************************************************************************************************
9  * @author Kurt Kramer *
10  * *
11  * Developer: Kurt Kramer *
12  * Date: 2007-10 *
13  * Place: Eastern Tropical Pacific (ETP) aboard the Seward Johnson. *
14  * *
15  * Purpose *
16  * ======= *
17  * My purpose in life is to provide a means for getting instrument data for a specified Plankton image. Given *
18  * an image file name we can derive the original SIPPER file that it came from and the scan line from within *
19  * this file where the image was extracted from. *
20  * *
21  * I will maintain InstrumentDataList objects; one for each SIPPER file that I am queried about. When I get *
22  * an image to look up If I do not already have its related 'InstrumentDataList' object I will: *
23  * 1) Look to see if a InstrumentData file already exists with this data. *
24  * 2) If no file available I will then go to the Original SIPPER file for the data and build a file that *
25  * can be used the next time rather than scanning the whole SIPPER file. *
26  * *
27  * - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - *
28  * *
29  * HOW TO USE: *
30  * "InitializePush" - Before using any functions call this static method. It will create the necessary data *
31  * structures needed. Then for each SIPPER image file that you need Instrument data for *
32  * call the static method "GetClosestInstrumentData". *
33  * *
34  * "InitializePop" - Before the application terminates call this static method to free up allocated memory. *
35  * *
36  * "GetClosestInstrumentData" - Call this static method to get the instrument data that is closest to the *
37  * SIPPER image file provided. *
38  * *
39  * "InitializePush" and "IniailizePop" replace "Initialize" and "CleanUp" respectively. The "Push" and *
40  * "Pop" describe the additional functionality provided. This way you no longer need to *
41  * worry if a caller has already initialized "IntrumenDdataFileNamager". *
42  * *
43  * *
44  * Sipper File Directories. *
45  * ======================== *
46  * This module will automatically look in a default directory "%PicesHomeDir%\SipperFiles". If you want it *
47  * to search other directories for SipperFiles you need to create a text file "SipperDirectoryList.txt" *
48  * in "%PicesHomeDir%\DataFiles\". Create one text line for each additional SubDirectory structure that you *
49  * want this module to search for SipperFiles. *
50  * *
51  ***************************************************************************************************************
52  @endcode
53  */
