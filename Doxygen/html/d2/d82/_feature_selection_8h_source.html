11 //* Description: *
12 //* ------------ *
13 //* Developed this application to test out the idea of using Feature Selection for each combination of *
14 //* binary classes. First started to play with this idea back in 2003 with a prvious version of this *
15 //* program called BeasmSearchSVM. *
16 //* *
17 //* Four Phases: *
18 //* 1) Parameter Tune for all Features. *
19 //* 2) Feature Selection for all classes combined. *
20 //* *
21 //* 3) Feature Selection for each binary class combination. (Parameters selected in phase 1 will be used)*
22 //* 4) Parameter Tuning for each binary class combination. *
23 //* *
24 //* Each phase must be completed before the next one can be started. *
25 //* *
26 //* Once all phases are completed a Validation run will be performed for each of the phases. That is *
27 //* he optimal parameters found by a given phase will be used to train a classifier. *
28 //* *
29 //* *
30 //* FeatureSelection: This module; main control. *
31 //* *
32 //* Processor: Created to perform a specific phase. Several processors will be created; one *
33 //* for each CPU that you want to utilize. The processor will schedule in a *
34 //* cooperative fashion all the jobs necessary to perform a given phase. A processor *
35 //* can be for either All-Classes or for a given binary combination of classes. *
36 //* *
37 //* BinaryJob: Used to control one single evaluation of parameters or features. When feature *
38 //* feature selection is started it will create a single 'BinaryJob' instance that *
39 //* will represent all features selected. Each expansion will then create more *
40 //* 'BinaryJob' instances. *
41 //* *
42 //* FeatureImpact: *
43 //* *
44 //* = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = = *
45 //* *
46 //* 1) the 1st Cmd Line parm is checked for certain commands. if one of them then it processes *
47 //* that command only. *
48 //* a. -restart *
49 //* b. -quit *
50 //* c. -FinalReport *
51 //* d. -Lock *
52 //* *
53 //* *
54 //* 2) If no 'FeatureSelection.Status' file exists then a new one is created and initialized to *
55 //* starting configuration. If one does exists then loads the current status of from it. When *
56 //* one does exist there is no need to provide any command lie parameters. *
57 //* *
58 //* *
59 //* 3) Feature Files are loaded and data is normalized. *
60 //* *
61 //* *
62 //* *
