Detailed Description

Supports the reading and writing of feature data from and to Pices feature data files.

*************************************************************************************************
* The PICES feature data file is a very simple format where the first few rows contain a *
* description of the rest of the data file. Any row that starts with "//" will be treated as *
* comment line. *
* First Row: Identifies the file as a feature Data File. An example on the next line. *
* "FEATURE_DATA_FILE Version 51 NumOfFeatures 88 NumOfImages 258" *
* Each field will be separated by a tab character. This line says that the *
* version number is 51. There are 88 features and 258 examples or feature vectors. *
* The version number gets incremented when ever there is a change in the way *
* that feature data is computed. *
* *
* Second Row: List all the fields in the feature data file. The first fields are feature *
* data. The first non feature field will be: "ClassName". There are several no *
* feature data fields. See the SipperSoftware documentation for details. *
*************************************************************************************************

FeatureDataReSink.

Synchronizes the contents of a feature data file with a directory of images.

Used with plankton applications to verify that feature file is up-to-date. Was specifically meant to work with training libraries, to account for images being added and deleted from training library. If there are no changes, then function will run very quickly.

A change in feature file version number would also cause all entries in the feature file to be recomputed. The feature file version number gets incremented whenever we change the feature file computation routine.

Parameters
[in]_dirNameDirectory where source images are located.
[in]_fileNameFeature file that is being synchronized.
[in]_unknownClassClass to be used when class is unknown
[in]_useDirectoryNameForClassNameif true then class name of each entry will be set to directory name.
[in]_databaseIf not NULL retrieves Instrument data from the PICES database table InstrumentData.
[in]_mlClassesList of classes
[in]_cancelFlagBoolean variable that will be monitored; if it goes True then will terminate processing right away and return to caller.
[out]_changesMadeIf returns as true then there were changes made to the feature file 'fileName'. If set to false, then no changes were made.
[out]_timeStampTimestamp of feature file.
[in]_logwhere to send diagnostic messages to.
Returns
- A ImageFeaturesList container object. This object will own all the examples loaded
Parameters
_cancelFlagwill be monitored, if set to True Load will terminate.

LoadInSubDirectoryTree.

Loads in the contents of a sub-directory tree.

Meant to work with SIPPER plankton images, it starts at a specified subdirectory and transverses all subdirectories. It makes use of FeatureDataReSink for each specific sub-directory.

Parameters
[in]_rootDirStarting directory.
[in,out]_mlClassesList of classes, any new classes in fileName will be added.
[in]_useDirectoryNameForClassNameif true set class names to sub-directory name. This happens because the user may manually move images between directories using the sub-directory name as the class name.
[in]_dataBaseIf not null retrieve Instrument data from the Instruments database table.
[in]_cancelFlagWill be monitored; if set to True Load will terminate.
[in]_rewiteRootFeatureFileIf true rewrite the feature file in the specified 'rootDir'. This feature file will contain all entries from all sub-directories below it.
[in,out]_logLogger where messages are to be written to.
Returns
- A ImageFeaturesList container object that will own all the examples loaded.
